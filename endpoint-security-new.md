Endpoint security vs Endpoint protection

**Endpoint Security** and **Endpoint Protection** are often used
interchangeably to describe the process of safeguarding devices that
connect to a network from cyber threats. While they are closely related,
there are some nuances between the two terms:

**Endpoint Security**

- **Broader term:** Encompasses the overall strategy of protecting
  endpoints, including security software, policies, and user education.

- **Key components:** Antivirus, anti-malware, intrusion detection, and
  other security tools.

- **Focus:** Protecting both individual devices and the network.

**Endpoint Protection**

- **Specific tools:** Refers to the individual security technologies
  used to safeguard endpoints.

- **Emphasis:** On the software solutions that protect devices from
  threats.

- **Component of endpoint security:** A subset of the broader endpoint
  security strategy.

**In essence:** Endpoint security is a comprehensive approach, while
endpoint protection focuses on the specific tools used within that
approach. Both are essential for protecting devices and networks from
cyber threats.

**Key Points:**

- **Objective:** Both aim to protect endpoints and the enterprise from
  security risks.

- **Deployment:** Endpoint security is typically on-premises, while
  endpoint protection is cloud-based.

- **Components:** Endpoint security includes antivirus, firewall,
  intrusion detection, and more.

- **Centralized management:** Endpoint security often involves
  centralized administration for enterprises.

- **Windows 10:** Even with built-in security, Windows 10 requires
  additional endpoint protection.

By understanding the distinctions between endpoint security and endpoint
protection, organizations can make informed decisions about their
security strategies and choose the appropriate tools to protect their
valuable assets.

served as one of the most accessible entry points for cyber threats —
and these threats have only grown more sophisticated over time. While
traditional antivirus software served as the initial line of defense,

**Endpoint Protection Platforms (EPP)**, which focus on preventing known
and unknown threats.

monitoring, investigation, and response to advanced threats that evade
initial protections. However, with the increasing complexity of IT
environments — spanning on-premises systems, cloud workloads, mobile
devices, IoT, and operational technology (OT) — a more comprehensive
solution has become necessary.

This has led to the emergence of **Extended Detection and Response
(XDR)**, a holistic approach that unifies data and threat response
capabilities across multiple security layers. As endpoints remain a
primary target — whether in a user's pocket, cloud, or server room —
robust, integrated protection is critical to defend against today’s
evolving threat landscape.

visibility and response.

- Leverages AI, machine learning, and threat intelligence to detect

- Functions as a **centralized security operations hub** for endpoint
  visibility.

that hunts them down and neutralizes them.*

EDR enables security teams to **trace attack sources**, understand

isolating endpoints or killing malicious processes — often **before
human intervention is required**.

more contextualized threat detection and response** capability.

**Key Features:**

- Ingests and analyzes telemetry across diverse sources.

- Uses AI/ML for correlation and behavioral analytics at scale.

How would you bypass AV?

Attackers have employed various techniques to bypass antimalware
software on Windows in the past. Here are some common methods:

- **Exploiting Antimalware Vulnerabilities:**

  - Antimalware software itself can have vulnerabilities that attackers
    can exploit. These vulnerabilities might allow attackers to disable
    specific features, bypass detection altogether, or even escalate
    privileges to gain more control over the system.

  - **Example:** In 2020, a vulnerability (CVE-2020-0601) in Microsoft
    Defender allowed attackers to learn about the locations excluded
    from Defender scans and plant malware there, effectively evading
    detection.

- **Using Obfuscation and Encryption:**

  - Attackers can obfuscate their malicious code by making it difficult
    for antimalware to read and understand its functionality. This can
    involve techniques like packing, encryption, or using complex
    algorithms.

  - **Example:** Packing tools compress and encrypt malicious code,
    making it appear like harmless data to some antimalware scanners.

- **Living Off the Land (LOL) Attacks:**

  - Attackers can leverage legitimate Windows tools and functionalities
    for malicious purposes. Since these tools are not inherently
    malicious, some antimalware programs might not flag them.

  - **Example:** Scripting languages like PowerShell are often used for
    legitimate tasks but can also be used to download malware or perform
    malicious actions. Antimalware might not detect a malicious
    PowerShell script if it relies solely on blacklisting known bad
    scripts.

- **Memory-based Attacks:**

  - Some malware operates primarily in memory without ever touching the
    disk. This makes it challenging for traditional signature-based
    antimalware to detect, as these scanners typically rely on
    identifying malicious code on the disk.

  - **Example:** Certain malware injects malicious code directly into
    legitimate processes running in memory, making detection more
    difficult.

- **Social Engineering:**

  - Social engineering tricks users into installing malware themselves.
    Phishing emails with malicious attachments or links are a common
    tactic. These emails might appear legitimate, luring users into
    clicking and unknowingly compromising their machines.

**Evolving Techniques:**

- Attackers are constantly developing new methods to bypass antimalware.
  Here are some additional points to consider:

  - **Machine Learning and Advanced Techniques:** Attackers are
    increasingly using machine learning to automate the creation of new
    malware variants that can evade detection.

  - **Fileless Malware:** As mentioned earlier, fileless malware that
    operates primarily in memory is a growing threat, posing challenges
    for traditional detection methods.

**Staying Protected:**

- **Keeping Antimalware Updated:** Antimalware vendors regularly release
  updates to address vulnerabilities and improve detection capabilities.
  Ensure you have automatic updates enabled for your antimalware
  software.

- **Security Awareness Training:** Educating users about social
  engineering tactics and best practices for safe browsing habits can
  significantly reduce the risk of falling victim to malware.

- **Layered Security:** A comprehensive security approach is crucial. In
  addition to antimalware, consider using firewalls, intrusion
  detection/prevention systems (IDS/IPS), and keeping your operating
  system and applications updated with the latest security patches.

BIOS vs. UEFI Firmware

Both BIOS and UEFI are firmware interfaces that manage the startup
process of a computer. However, UEFI is a more modern and advanced
version that offers several advantages over BIOS

**Functionality:**

- **BIOS (Basic Input/Output System):** A legacy interface with limited
  functionality. It initializes basic hardware components, loads the
  operating system, and provides low-level control.

- **UEFI (Unified Extensible Firmware Interface):** A more advanced and
  feature-rich interface that replaces BIOS. It offers faster boot
  times, improved security features, and greater hardware support.

**Technical Specifications:**

- **Mode:** BIOS operates in 16-bit mode, limiting its capabilities.
  UEFI runs in 32-bit or 64-bit mode, allowing for more complex
  operations.

- **User Interface:** BIOS has a text-based interface that can be
  difficult to navigate. UEFI offers a graphical user interface (GUI)
  with mouse support, making it more user-friendly.

- **Storage Support:** BIOS has limitations with hard drives exceeding
  2.2 terabytes (TB) due to the Master Boot Record (MBR) partitioning
  scheme. UEFI utilizes the GUID Partition Table (GPT), allowing for
  drives much larger than 2.2 TB.

**Security:**

- **Limited Security:** BIOS offers minimal security features.

- **Enhanced Security:** UEFI provides features like Secure Boot, which
  verifies the legitimacy of the operating system before booting,
  protecting against unauthorized software.

**Other Advantages of UEFI:**

- **Faster Boot Times**: UEFI can boot a computer significantly faster
  than BIOS due to its advanced features and initialization processes.

- **Driver Support**: UEFI can load drivers directly, whereas BIOS
  relies on the operating system to load them.

- **Network Support:** Some UEFI implementations offer basic network
  capabilities, allowing for network-based updates or troubleshooting.

# Intrusion Detection and Prevention System (IDPS)

IDPS stands for Intrusion Detection and Prevention Systems (IDPS)

IDPS monitors a single host/network for suspicious activity by comparing
system snapshots.

**Key Features and Functions:**

- **Monitoring:** IDPS continuously monitors networks or systems for
  malicious activity or policy violations.

- **Analysis:** It analyses network traffic to detect known threats and
  suspicious behaviours.

- **Detection:** It identifies external influences attempting to gain
  unauthorized access to software.

- **Comparison:** IDPS compares network traffic to signatures,
  behaviours, or rules.

**Intrusion Prevention Systems (IPS):**

- **Prevention:** IPS not only detects but also blocks malicious
  activity.

- **Configuration:** It can be configured to deny or redirect traffic.

- **Database:** Requires a regularly updated database of threats.

- **Visibility and Detection:** Allows for monitoring network and
  endpoint activity.

- **Configuration:** Enables customization of alerts and traffic
  profiling.

- **Continuous Monitoring:** Provides ongoing surveillance of network
  and endpoints.

- **Incident Detection:** Detects security breaches and DDoS attacks.

- **Anomaly Detection:** Identifies unusual or suspicious network
  traffic patterns.

- **Threat Tracking:** Monitors for suspicious and malicious behaviours.

- **SQL Injection Detection:** Recognizes and alerts on SQL injection
  attempts.

- **Alert Generation:** Produces notifications when threats are
  detected.

- **XSS Detection:** Identifies and alerts on cross-site scripting
  attacks.

- **Early Detection:** Provides timely identification of threats.

- **Proactive Response:** Enables swift action to mitigate threats.

- **Deterrent Effect:** Discourages attackers due to the presence of an
  IDPS.

- **Enhanced Security:** Improves overall network and endpoint security.

- **Precise Localization:** Pinpoints the source of security incidents.

# IDPS Alerts

  than missing real threats).

- False negative: IDS fails to detect actual intrusion (most serious
  issue).

**Why the IT and Security teams do not like agents?**

IT and Security teams often have concerns about endpoint security agents
for a few reasons:

**Increased Complexity and Resource Consumption:**

- **Management Overhead:** Managing and maintaining multiple security
  agents from different vendors can be complex and time-consuming for IT
  teams.

- **Resource Drain:** Security agents can consume system resources like
  CPU, memory, and storage, potentially slowing down devices. This can
  be a concern for older machines or those with limited resources.

**Potential Security Risks:**

- **Attack Surface:** Each agent introduces a new potential
  vulnerability that attackers can exploit.

- **Supply Chain Attacks:** A compromised agent from a trusted vendor
  can introduce malware across a network.

- **False Positives and Negatives:** Security agents can generate false
  positive alerts (flagging harmless applications as threats) and false
  negatives (missing actual threats). This can overload security teams
  and reduce overall effectiveness.

**Visibility and Control Issues:**

- **Limited Visibility:** Some agents might not provide a comprehensive
  view of all security threats across the entire network.

- **Agent Silos:** Multiple agents with different interfaces can make it
  difficult to get a holistic view of security posture.

- **Agent Conflicts:** Different agents might interfere with each other,
  causing compatibility issues or unexpected behaviour.

**Here are some alternative approaches that IT and Security teams might
prefer:**

- **EDR (Endpoint Detection and Response):** EDR solutions focus on
  continuous monitoring, threat detection, investigation, and response,
  offering a more proactive approach.

- **Cloud-Based Security:** Cloud-based security solutions can offer
  centralized management, reduced resource consumption on devices, and
  potentially better scalability.

- **OS-Level Security:** Modern operating systems often have built-in
  security features that can be quite effective, especially when
  combined with good security practices.

**However, endpoint security agents still have a role to play.** They
can be valuable tools when used strategically, particularly for:

- **Legacy Systems:** For older systems that lack robust built-in
  security, agents can provide an additional layer of protection.

- **Targeted Deployments:** Agents can be deployed on specific devices
  that require extra security due to their role or the data they handle.

- **Specialized Features:** Some agents offer unique features like
  application control or data encryption that might not be available
  elsewhere.

Ultimately, the decision of whether to use endpoint security agents
depends on a careful evaluation of the organization's specific needs and
security posture.

- **Cons:**

  - **Larger Attack Surface:** Due to its widespread use, Windows is a
    more attractive target for cybercriminals.

  - **Prevalent Software Vulnerabilities:** The sheer volume of software
    on Windows can lead to more frequent vulnerabilities.

  - **Default Admin Access:** Windows accounts often have administrative
    privileges by default, increasing potential damage from malware.

**<u>Linux:</u>**

- **Pros:**

  - **Open-Source Development:** The open-source nature allows anyone to
    scrutinize the code, potentially leading to faster identification
    and fixing of vulnerabilities.

  - **User Permissions:** By default, Linux users don't have full
    administrative access, limiting the damage potential of malware.

  - **Smaller Attack Surface:** Linux has a lower market share compared
    to Windows, making it a less attractive target for attackers.

- **Cons:**

  - **Fewer Security Software Options:** There are fewer commercially
    available security solutions compared to Windows.

  - **Steeper Learning Curve:** Linux can be more complex to manage and
    secure for non-technical users.

  - **Potential Lack of Awareness:** Because it's less common, some
    users might be less aware of potential security threats on Linux.

**Overall:**

- For ease of use and wider security software options, Windows might be
  preferable, especially for non-technical users who can leverage a
  larger security community and user-friendly features.

- However, if security is your top priority and you're comfortable with
  a more technical system, Linux offers a smaller attack surface,
  stronger permission controls, and the potential benefit of open-source
  development.

**Remember:** Regardless of the OS, security best practices like keeping
software updated, using strong passwords, and being cautious online are
crucial for both Windows and Linux.

# Endpoint Security Architecture

 

Here are the key components of endpoint architecture in Windows and
Linux environments:

- **Operating System (OS):**

  - Windows: The dominant OS for desktops and laptops, with versions
    like Windows 10, 11, and Server.

  - Linux: Open-source OS popular for servers, cloud computing, and
    embedded systems, with distributions like Ubuntu, Red Hat, and
    Debian.

- **Hardware**:

  - CPU: Central processing unit, the "brain" of the device.

  - RAM: Random access memory, for storing data in active use.

  - Storage: Hard drives (HDDs) or solid-state drives (SSDs) for storing
    data persistently.

  - Network adapters: Ethernet, Wi-Fi, or Bluetooth cards for network
    connectivity.

  - Peripherals: Input devices (keyboard, mouse), output devices
    (monitor, printer), and other connected devices.

- **File System:**

  - Windows: NTFS (New Technology File System), a proprietary file
    system designed for Windows.

  - Linux: Ext4 (Extended File System 4), a common open-source file
    system for Linux.

- **Users and Groups:**

  - Accounts: Individual user accounts with specific permissions and
    access rights.

  - Groups: Collections of users with shared permissions and access
    levels.

- **Processes and Services:**

  - Processes: Executable programs running on the system.

  - Services: Background processes that perform specific tasks, often
    system-related.

- **Permissions and Security:**

  - Access control: Restricting access to files, folders, and resources
    based on user accounts and groups.

  - Firewalls: Software or hardware barriers that filter network traffic
    to protect against unauthorized access.

  - Antivirus and anti-malware: Software that detects and prevents
    malicious software infections.

- **Applications and Software:**

  - Productivity suites: Office software, email clients, web browsers,
    etc.

  - Development tools: Compilers, interpreters, debuggers, code editors.

  - Utilities: System maintenance, file management, disk management,
    etc.

- **Configuration and Management:**

  - Settings: System-wide configuration options for hardware, software,
    and security.

  - Tools: Utilities for managing users, groups, processes, services,
    and security settings.

- **Networking**:

  - Protocols: TCP/IP, HTTP, DNS, DHCP, etc.

  - Stack: Software that implements network protocols and enables
    communication over networks.

  - Interfaces: Physical or virtual adapters for connecting to networks.

- **Logging and Monitoring:**

  - Event logs: Records of system events, errors, and warnings.

  - Performance metrics: Data on CPU usage, memory usage, disk activity,
    network traffic, etc.

  - Security logs: Audit trails of user activity and security events.

- **Updates and Patches:**

  - Security patches: Fix vulnerabilities and address security threats.

  - Feature updates: Add new features and functionality.

 

# Endpoint Security Strategy

- specify security baselines for server and client endpoints

- specify security requirements for servers, including multiple
  platforms and operating systems

- specify security requirements for mobile devices and clients,
  including endpoint protection, hardening, and configuration

- specify requirements to secure Active Directory Domain Services

- design a strategy to manage secrets, keys, and certificates

- design a strategy for secure remote access

**<u>Understanding the Scope</u>**

Before diving into the strategy, it's crucial to define the scope of the
environment. This includes:

- **Endpoint types:** Servers, desktops, laptops, mobile devices, IoT
  devices

- **Operating systems:** Windows, macOS, Linux, iOS, Android

- **Network topology:** On-premises, cloud, hybrid

- **Industry regulations:** Compliance requirements (e.g., PCI DSS,
  HIPAA, GDPR)

Security Baselines

A security baseline is a standardized configuration for systems and
applications. It serves as a foundation for security controls.

Server Endpoints:

- **Operating system patches:** Regularly updated to the latest patch
  level.

- **Application whitelisting:** Only approved applications allowed to
  run.

- **Network configuration:** Firewalls, intrusion detection/prevention
  systems (IDS/IPS), network segmentation.

- **Account management:** Strong password policies, privileged account
  management, least privilege principle.

- **Security software:** Antivirus, anti-malware, endpoint detection and
  response (EDR).

Client Endpoints:

- **Operating system patches:** Regular updates.

- **Application controls:** Similar to servers, but with more focus on
  user behavior.

- **Data protection:** Encryption for sensitive data, data loss
  prevention (DLP).

- **Remote access controls:** VPN, MFA, access controls.

- **Mobile device management (MDM):** For mobile devices, ensuring
  compliance and security.

Security Requirements for Servers

- **Platform-specific hardening:** Tailored security measures for
  different platforms (Windows, Linux, etc.).

- **Vulnerability management:** Regular scanning and patching.

- **Logging and monitoring:** Comprehensive logging and real-time
  monitoring for threats.

- **Intrusion detection and prevention:** Deploy IDS/IPS solutions.

- **Security information and event management (SIEM):** Centralized log
  management and analysis.

- **Incident response plan:** Define procedures for handling security
  incidents.

Security Requirements for Mobile Devices and Clients

- **Endpoint protection platform (EPP):** Comprehensive security suite
  for malware protection.

- **Endpoint detection and response (EDR):** Advanced threat detection
  and response capabilities.

- **Mobile device management (MDM):** Enforce security policies, remote
  wipe, and data protection.

- **Application whitelisting:** Restrict applications to approved ones.

- **Data encryption:** Encrypt sensitive data both at rest and in
  transit.

- **User education:** Train users about security best practices.

Securing Active Directory Domain Services

- **Password policies:** Strong password requirements, password
  expiration, and lockout policies.

- **Group policy objects (GPOs):** Configure security settings
  centrally.

- **Domain controller hardening:** Implement security best practices for
  domain controllers.

- **Account management:** Regular audits, privileged account management,
  and least privilege principle.

- **Active Directory Federation Services (AD FS):** For federated
  identity management.

Managing Secrets, Keys, and Certificates

- **Secret management solution:** Use a dedicated tool to store and
  manage secrets securely.

- **Key management infrastructure (KMI):** Manage cryptographic keys for
  encryption and decryption.

- **Certificate management system:** Automate certificate lifecycle
  management.

- **Access controls:** Restrict access to secrets, keys, and
  certificates.

- **Regular audits and rotation:** Ensure secrets and keys are rotated
  regularly.

Secure Remote Access

- **Virtual private network (VPN):** Encrypted connection for remote
  access.

- **Multi-factor authentication (MFA):** Require multiple forms of
  authentication.

- **Access controls:** Restrict access based on user roles and
  permissions.

- **Session monitoring:** Record and monitor remote sessions for
  security.

- **Endpoint security checks:** Verify endpoint compliance before
  granting access.

Additional Considerations:

- **Regular security assessments:** Conduct vulnerability assessments
  and penetration testing.

- **Employee training:** Educate employees about security best
  practices.

- **Incident response plan:** Develop a comprehensive incident response
  plan.

- **Business continuity and disaster recovery:** Plan for system
  failures and data loss.

- **Compliance:** Adhere to relevant industry regulations and standards.

By implementing these strategies and continuously monitoring and
adapting to the evolving threat landscape, organizations can
significantly enhance their endpoint security posture.

**✅ Endpoint Security Checklist for Small and Medium Businesses**

**🔒 Preventing Initial Compromise**

The best defense against cyberattacks is to prevent attackers from
gaining access in the first place.

**🔧 Vulnerable Software**

- **Patch What You Can**\
  Apply security patches as soon as vulnerabilities are disclosed.
  Implement a patch management process to assess, test, and deploy
  updates.

- **Isolate What You Can’t Patch**\
  If patching isn’t feasible, restrict network access to vulnerable
  systems to minimize exposure.

**🌐 Exposed Ports & Services**

- **Secure Remote Desktop Protocol (RDP)**\
  Don’t expose RDP to the internet. Use firewalls, RDP Gateway, and
  Network Level Authentication (NLA). Change the default RDP port (TCP
  3389).

- **Secure Server Message Block (SMB)**\
  Disable SMBv1. Restrict SMB with firewalls to prevent threats like
  WannaCry and EternalBlue.

**📧 Email**

- **Block Common Malicious Attachments**\
  Block file types like .EXE, .BAT, .JS, .VBS, .ZIP, .SFX, .7z, .DOCX,
  and .PDF to reduce email-borne threats.

- **Conduct User Awareness Training**\
  Train users to recognize phishing and social engineering tactics that
  rely on deception.

**🌍 Browsers**

- **Use Ad-Blockers**\
  Malvertising can spread malware even from legitimate sites.
  Ad-blockers help reduce this risk.

**📄 Microsoft Office**

- **Enforce Stricter Macro Controls**\
  Block macros in files downloaded from the internet to prevent
  script-based malware execution.

- **Disable “Update Automatic Links at Open”**\
  Prevent abuse of the DDE feature (now disabled by default).

- **Disable OLE Packages**\
  Disable Microsoft’s Object Linking and Embedding (OLE) feature when
  not needed due to historical abuse.

**🛡️ Mitigating Post-Exploitation Techniques**

Once attackers gain access, they often use built-in tools and fileless
techniques. Defend proactively:

**💻 PowerShell**

- **Disable PowerShell (if not required)**\
  It’s powerful—and dangerous in the wrong hands.

- **Use the Latest Version**\
  Newer versions include enhanced logging and security.

- **Block Unsigned Scripts**\
  This deters basic attack attempts and raises visibility.

- **Enable Constrained Language Mode**\
  Restricts PowerShell’s functionality to limit malicious usage.

- **Enable and Monitor Logging**\
  Use extended logging to capture attack activity (expect large logs).
  Tools like PowerShell Method Auditor help process logs.

**⚙️ Windows Management Instrumentation (WMI)**

- **Create Defensive WMI Subscriptions**\
  Use WMI for detection and response (examples
  [here](https://chatgpt.com/c/683883ae-f740-8013-9866-4baab4ef4102) and
  [here](https://chatgpt.com/c/683883ae-f740-8013-9866-4baab4ef4102)).

- **Restrict Remote WMI if Unnecessary**\
  Use a fixed port and block externally if WMI isn’t needed remotely.

**🧰 Application Control**

- **Enforce AppLocker Rules**\
  Restrict execution of unauthorized executables, DLLs, and scripts.

- **Harden AppLocker**\
  Learn bypass techniques and create counter-rules to strengthen
  defenses.

**🔐 Least Privilege & Access Controls**

- **Apply Least Privilege Principles**\
  Users should have only the access they need. Use Microsoft's Just
  Enough Administration.

- **Set UAC to Highest Enforcement**\
  "Always notify" minimizes silent privilege elevation.

- **Enable Admin Approval Mode**\
  Enforce UAC for the built-in Administrator account.

- **Remove Users from Local Admin Groups**\
  Prevents privilege escalation.

- **Disable Credential Caching**\
  Avoid storing credentials for network authentication.

- **Prevent Credential Overlap**\
  Unique credentials across systems reduce lateral movement risks.

- **Don’t Stay Logged Into Remote Systems**\
  Open sessions can be hijacked.

- **Disable Anonymous Access on NFS and FTP**\
  Prevent unauthorized file share access and lateral spread.

- **Use Strong Passwords**\
  A basic but still often-overlooked defense.

- **Enable Two-Factor Authentication (2FA)**\
  Blocks unauthorized access, even with stolen passwords.

- **Configure Account Lockout Policies and Login Delays**\
  Slows down or blocks brute force attacks.

**🕵️ Monitoring & Detection**

- **Monitor Registry Changes**\
  Detect persistence techniques like hidden scripts.

- **Watch for Suspicious WMI Activity**\
  Monitor with defensive subscriptions.

- **Track Scheduled Task Creation**\
  Scheduled tasks are commonly used for persistence.

- **Observe Suspicious Processes & API Calls**\
  Use tools like Sysinternals Process Explorer and Get-InjectedThreads.

- **Detect CREATE_SUSPENDED Flag in New Processes**\
  Indicates process hollowing—used in stealthy attacks.
